## Screenshots Requirements

### Google Play Store
- **Minimum:** 2 screenshots
- **Recommended:** 4-8 screenshots
- **Format:** PNG or JPEG
- **Size:**
  - Phone: 16:9 aspect ratio
  - Recommended: 1080 x 1920 pixels
- **Tablet:** 1200 x 1920 pixels (optional but recommended)

### Apple App Store
- **Minimum:** 1 screenshot per device type
- **Recommended:** 3-10 screenshots
- **Formats:**
  - iPhone 6.5": 1242 x 2688 pixels
  - iPhone 5.5": 1242 x 2208 pixels
  - iPad Pro 12.9": 2048 x 2732 pixels

### Screenshot Ideas

1. **Home Screen / Topic List**
   - Show list of topics with review dates
   - Highlight due topics
   - Show folders organization

2. **Topic Editor**
   - Markdown editor in action
   - Show rich formatting
   - Clean, focused interface

3. **Calendar View**
   - Monthly calendar with review markers
   - Show upcoming reviews
   - Visual scheduling

4. **Statistics Dashboard**
   - Review count
   - Streak tracker
   - Progress charts

5. **Topic Detail / Reader**
   - Rendered markdown
   - "Mark as Reviewed" button
   - Clean reading experience

6. **Settings / Google Drive**
   - Notification settings
   - Cloud backup option
   - Customization options

### Screenshot Tips
- Use high contrast text
- Show real, meaningful content (not Lorem Ipsum)
- Add captions/text overlays highlighting key features
- Use consistent branding colors
- Show the app in use, not empty states
- Demonstrate value in first screenshot

## Release Build Commands

### Android
```bash
# Build release APK
flutter build apk --release

# Build release App Bundle (recommended for Play Store)
flutter build appbundle --release

# Location:
# APK: build/app/outputs/flutter-apk/app-release.apk
# AAB: build/app/outputs/bundle/release/app-release.aab
```

### iOS
```bash
# Build release IPA
flutter build ios --release

# Then use Xcode to archive and upload to App Store
```

- **MAJOR:** Breaking changes
- **MINOR:** New features (backward compatible)
- **PATCH:** Bug fixes
